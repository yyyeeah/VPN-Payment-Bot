# VPN Payment Bot

Телеграм-бот для ручной продажи VPN-ключей. Он не трогает ключи вообще, а берет на себя только платежную рутину:

- напоминает об оплате за 1 день до окончания подписки и в день окончания
- шлет ежедневные напоминания о просрочке и клиенту, и администратору
- принимает чеки от клиентов
- пересылает чеки в админ-чат на ручную проверку
- позволяет вручную выставлять и продлевать срок подписки
- хранит число устройств по каждому клиенту и считает сумму как `150₽ * количество устройств`
- позволяет задать для клиента отдельное имя, которое видит только администратор
- позволяет отключать прием чеков для отдельных клиентов и удалять клиентов из активной базы
- помечает дубли чеков по `file_unique_id`
- ведет историю изменений по клиентам
- показывает каждому клиенту его `client_id`, который можно скопировать в одно нажатие

## Как устроен бот

### Для клиента

1. Клиент впервые пишет боту.
2. Бот показывает только `ID клиента` и кнопку `Скопировать ID`.
3. Клиент отправляет этот ID вам.
4. Вы привязываете клиента у себя через `/link <client_id> <DD.MM.YYYY|+30|+1m>`.
5. После привязки клиент получает клиентские команды и может отправлять чеки.
6. Клиент нажимает `/pay`, бот показывает реквизиты, а следующим сообщением принимает чек и отправляет его вам на проверку.
7. Клиент может отдельно открыть `/price`, чтобы посмотреть число устройств, итоговую сумму и тариф `150₽ за 1 устройство` без реквизитов.

В сообщениях про оплату бот пишет:

- сумму в формате `300₽ (2 устройства)`
- строку реквизитов из `PAYMENT_DESTINATION_TEXT` в `.env`

Важно: бот не добавляет всех подряд в список активных клиентов. В управляемый список попадают только те, кому вы реально задали срок подписки.

### Для администратора

1. Настройте `ADMIN_CHAT_ID` в `.env` или назначьте админ-чат через `/admin <ADMIN_CODE>`.
2. В админ-чате открывается отдельное меню команд администратора.
3. Получайте чеки в админ-чате.
4. Используйте кнопки `+30 дней`, `+90 дней`, `Свой срок` или команды.
5. Если клиент еще не привязан, попросите его `client_id` и выполните `/link`.

## Команды администратора

- `/admin <code>`: назначить текущий личный чат админским, если `ADMIN_CHAT_ID` не задан
- `/clients`: список привязанных клиентов с датами
- `/client <client_id|user_id>`: подробная карточка клиента с последними чеками и историей изменений
- `/pending`: список чеков на проверке
- `/link <client_id> <DD.MM.YYYY|+30|+1m>`: привязать клиента по его ID и сразу задать дату или срок
- `/setexpiry <client_id|user_id> <DD.MM.YYYY|+30|+1m>`: поставить точную дату или срок
- `/extend <client_id|user_id> <30|1m>`: продлить подписку на дни или месяцы
- `/setdevices <client_id|user_id> <count>`: изменить число устройств клиента и сумму
- `/setname <client_id|user_id> <имя|->`: задать или очистить имя клиента для администратора
- `/mutereceipts <client_id|user_id> <on|off>`: выключить или включить прием чеков
- `/deleteclient <client_id|user_id>`: убрать клиента из активных
- `/broadcast`: ответом на сообщение в админ-чате разослать его всем известным пользователям
- `/reject <причина>`: ответом на карточку чека отклонить оплату
- `/cancel`: отменить ввод срока после кнопки `Свой срок`
- `/runreminders`: вручную запустить проверку напоминаний

Ответом на карточку чека можно быстро использовать:

- `/setexpiry 30.04.2026`
- `/setexpiry +1m`
- `/extend 30`
- `/extend 2m`
- `/reject дубль`

Для массовой рассылки:

- отправьте в админ-чат сообщение нужного формата
- ответьте на него командой `/broadcast`
- бот скопирует это сообщение всем пользователям, которые уже открывали с ним личный чат

## Команды клиента

До привязки клиенту доступны только:

- `/start`
- `/id`

После привязки доступны:

- `/status`
- `/price`
- `/pay`
- `/id`
- `/whoami`

## Запуск

1. Создайте бота через BotFather и получите токен.
2. Скопируйте `.env.example` в `.env`.
3. Заполните как минимум `BOT_TOKEN` и `PAYMENT_DESTINATION_TEXT`.
4. Вариант A: сразу укажите `ADMIN_CHAT_ID`.
5. Вариант B: оставьте `ADMIN_CHAT_ID` пустым, задайте `ADMIN_CODE`, запустите бота и напишите ему `/admin <ADMIN_CODE>`.

Установка и запуск:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
python main.py
```

## Что важно помнить

- Чтобы бот мог писать клиенту напоминания, клиент должен хотя бы один раз открыть диалог с ботом.
- Даты подписки хранятся как календарные даты в часовом поясе из `TIMEZONE`.
- Напоминание по умолчанию уходит за 1 день до окончания подписки и в день окончания.
- После просрочки напоминание уходит каждый день и дублируется в админ-чат.
- Защита от дублей включена, поэтому ежедневную проверку можно запускать безопасно.
- База данных по умолчанию: SQLite.
- Тексты напоминаний можно переопределить через `REMINDER_*_TEMPLATE` и `ADMIN_OVERDUE_REMINDER_TEMPLATE` в `.env`.
